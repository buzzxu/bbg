# /plan -- Implementation Planning

Read and follow the agent instructions in `agents/planner.md` and the skill workflow in `skills/tdd-workflow/SKILL.md`.

## Steps

1. Analyze the current codebase structure and understand existing patterns
2. Identify the scope of changes required
3. Break down the implementation into small, testable steps
4. Consider edge cases and error handling
5. Review security implications (see `rules/common/security.md`)
6. Create the implementation plan with clear milestones
