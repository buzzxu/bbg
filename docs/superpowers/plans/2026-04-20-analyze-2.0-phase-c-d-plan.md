# Analyze 2.0 Phase C/D Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deliver the next two Analyze 2.0 phases after the Phase A/B substrate is in place: first add the Socratic interview engine behind a safe feature flag, then upgrade `bbg analyze` into a one-command AI-first experience with improved tool routing and auto-launch behavior.

**Architecture:**

- **Phase C** is an internal engine upgrade. It introduces evidence-driven questioning while preserving the legacy interview path as fallback.
- **Phase D** is a UX and orchestration upgrade. It preserves the existing CLI entry point and handoff model, but shifts the preferred user journey toward direct AI-context execution or automatic launch.

**Tech Stack:** TypeScript, Vitest, current analyze pipeline, existing evidence/lifecycle substrate from Phase B, current CLI and handoff runtime, existing agent runner configuration.

## Phase C/D Sequencing Notes

Recommended execution order:

1. C1
2. C2
3. C3
4. C4
5. C5
6. C6
7. D1
8. D2
9. D3
10. D4
11. D5
12. D6

Why this order:

- Phase C should stay mostly pure and internal until the engine is coherent
- the feature-flagged integration happens only after the pure pieces are stable
- Phase D should begin with behavior tests so UX changes remain intentional
- auto-launch and copy improvements come after the expected flow is locked

---

## Scope Guardrails

Do **not** include in Phase C/D:

- Hermes candidate ingestion and promotion
- runtime verification feedback loops
- global knowledge promotion
- fully autonomous LLM self-interview unless it falls out with very low risk
- adding a new public command family for analyze
- adding permanent user-facing flags for AI mode selection

Phase C/D is complete when the new interview engine can run safely behind a flag and the analyze command provides a clearly AI-first but still low-learning-cost user experience.
