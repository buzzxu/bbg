# BBG Harness 9.5 — Master Implementation Plan

> **Source of Truth**: `docs/superpowers/specs/2026-04-03-harness-9.5-design.md` (1629 lines, approved)

**Goal:** Upgrade BBG-generated AI Harness from 8/10 to 9.5/10 completeness across 8 capability modules, plus MBTI deep personality modeling for all 25 agents.

**Architecture:** BBG is a GENERATOR — all new capabilities are governance assets deployed to target projects via `bbg init`. BBG's own `src/` code handles analysis, rendering, and installation. This master plan coordinates 8 independent phase plans.

**Key Constraint:** BBG generates content for OTHER projects. Do NOT confuse BBG's own functionality with what it generates for target projects.

## Shared Infrastructure

### SQLite Foundation (established in Phase 2, used by all subsequent phases)

- Database file: `.bbg/telemetry.db` (in target project)
- Initialization: schema scripts are deployed to `.bbg/scripts/` and initialized by project workflow commands/hooks according to each phase plan (idempotent SQL via `CREATE TABLE IF NOT EXISTS`)
- Each phase adds its own SQL file: `.bbg/scripts/<module>-schema.sql`
- Total: 24 tables, 14 views across all modules

### Governance Manifest Changes (cumulative)

- `src/templates/governance.ts` — each phase registers new templates
- `tests/unit/templates/governance.test.ts` — count assertions updated per phase
- Each phase must update the total counts in the test file

## TDD Strategy

Every phase follows the same pattern:

1. Write failing test for governance manifest count changes
2. Create the governance content files (agents, skills, commands, etc.)
3. Register in `governance.ts`
4. Update test assertions
5. Run `npm test` + `npm run build`
6. Commit
