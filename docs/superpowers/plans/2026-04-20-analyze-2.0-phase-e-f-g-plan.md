# Analyze 2.0 Phase E/F/G Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Complete the Analyze 2.0 rollout by connecting analyze outputs into the Hermes candidate lifecycle, adding task and verification feedback loops, and finally switching the new analyze path to default while safely deprecating the legacy path.

**Architecture:**

- **Phase E** makes analyze a first-class producer of Hermes-ingestable candidate knowledge.
- **Phase F** closes the loop by letting runtime work and verification confirm, refine, or contradict prior analyze knowledge items.
- **Phase G** makes the new model the default only after the substrate, UX, and feedback loops are proven stable.

**Tech Stack:** TypeScript, Vitest, current Hermes command/runtime layer, existing analyze knowledge outputs from Phases A-D, current verify command and runtime context model, existing wiki and Hermes database conventions.

- rollout only happens after the full loop is closed and measurable

## Scope Guardrails

Do **not** include in Phase E/F/G:

- automatic promotion of local candidate knowledge to global BBG assets
- speculative confidence boosts without explicit evidence links
- new user-facing analyze command families
- permanent exposure of rollout/fallback internals to normal users
- silent overwriting of contradicted knowledge items

Phase E/F/G is complete when analyze outputs fully participate in the Hermes lifecycle, runtime work can strengthen or challenge prior understanding, and Analyze 2.0 is safely established as the default path.
