## 2. Executive Summary

Analyze 2.0 is a proposal to evolve `bbg analyze` from a mostly deterministic repository scanning pipeline into an **AI-native project understanding system**.

The target system should:

- understand technical and business structure more deeply
- use evidence-driven Socratic questioning when the business is unclear
- run deep analysis in AI context while keeping a one-command UX
- write durable reusable knowledge into `.bbg/knowledge/` and `docs/wiki/`
- feed Hermes as a governed learning pipeline
- let future task execution and verification strengthen or contradict prior understanding

The design goal is not to add more command complexity. The design goal is to make `bbg analyze` smarter while keeping it simple to use.

## 6. Proposal Boundaries

Analyze 2.0 does **not** propose:

- adding a new family of analyze commands for normal users
- forcing users to manually understand handoff internals
- replacing JSON knowledge files with database-only storage
- auto-promoting speculative AI output to global BBG assets
- skipping review and verification in the knowledge lifecycle

Analyze 2.0 **does** propose:

- evidence-backed AI-driven analysis
- minimal-friction UX
- durable structured project knowledge
- Hermes-governed knowledge refinement
- runtime and verification feedback loops

## 7. Recommended Review Questions

When reviewing this package, these are the most important questions to resolve:

1. Is the one-command UX rule non-negotiable?
2. Is the additive data model sufficient, or is a deeper schema migration needed later?
3. Which knowledge item types should be candidate-worthy by default?
4. What is the rollout policy for switching the new path to default?
5. Which parts of autonomous/self-interview should wait until after the deterministic Socratic engine is stable?
6. How conservative should confidence uplift and contradiction handling be?
